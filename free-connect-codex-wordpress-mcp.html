Free - Connect Codex With WordPress via MCP

Automate your WordPress website using Codex — OpenAI's free AI coding agent. Create blog posts, run SEO tasks, and manage pages through natural language chat. Free to start, no coding required.

What You Can Do with Codex + WordPress MCP

Codex is an AI coding agent from OpenAI — a direct competitor to Claude AI that's free to start (paid plan from $8/month). Once connected to your WordPress site via MCP, Codex becomes a powerful automation layer that lets you manage your entire website through natural language chat:

  • Create fully formatted, SEO-friendly blog posts directly inside WordPress from a single chat message
  • Update post meta titles and meta descriptions for better search engine rankings
  • Create new WordPress pages and layouts without touching the dashboard
  • Audit all existing posts and get suggestions on how to rank better on Google, Bing, and other search engines
  • Analyze your entire content library and identify gaps or optimization opportunities
  • Manage WooCommerce products, categories, and store content through plain English
  • Explore and test WordPress capabilities without hiring a developer or writing code

Codex connects to WordPress via the Model Context Protocol (MCP) — the same open standard used by Claude AI. The free WSP WordPress MCP plugin exposes your WordPress REST API to Codex in a structured, permission-gated way. You control exactly what Codex can and cannot do on your site.

Codex vs Claude AI for WordPress: Codex is free to start and plans from $8/month make it accessible for beginners. Claude AI offers higher output quality but starts at $20/month. Both use the same WordPress MCP plugin — so you can try Codex first and switch to Claude later without changing your WordPress setup.

How to Connect Codex with WordPress: Step-by-Step

01

Download the MCP Adapter Plugin

The MCP Adapter bridges your WordPress REST API with Codex's MCP protocol. Go to the MCP Adapter GitHub repository, click Releases, and download the latest .zip file.

02

Download the WSP WordPress MCP Plugin

This free, open-source plugin by WebSensePro exposes your WordPress abilities to Codex. Download the latest release from the WSP WordPress MCP GitHub repository. MIT licensed, no account needed.

03

Install and Activate Both Plugins in WordPress

In your WordPress dashboard, go to Plugins → Add Plugin → Upload Plugin. Upload and activate the MCP Adapter first, then the WSP WordPress MCP plugin. After activation, a new "MCP" menu item appears in your sidebar.

04

Configure Which Abilities Codex Can Access

Click MCP → Settings. Toggle on the abilities you want Codex to use: create posts, read posts, delete posts, manage pages, WooCommerce products, taxonomy, comments, media, users, and site settings. Toggle off anything you want to restrict.

05

Copy the Codex Config Snippet

Go to MCP → Config Files and click "Copy to Clipboard" under the Codex section. This TOML-formatted snippet contains your WordPress URL and MCP endpoint — it's what tells Codex where your site lives.

06

Generate a WordPress Application Password

Go to Users → All Users → Edit your admin user. Scroll to Application Passwords, create a new one (name it "Codex"), copy it, and replace the replace with your application password placeholder in the config snippet. Treat this like a password — do not share it.

07

Paste the Config into Codex's config.toml and Restart

Open Codex → Settings → Configuration → Open config.toml. Scroll to the bottom of the file and paste the snippet you copied from the plugin. Save the file, then fully close and reopen Codex. Go back to Settings — under MCP Server you should now see the WordPress MCP server listed alongside your existing servers.

08

Test It — Ask Codex to Create a Blog Post

Open a new Codex chat and type: "Please create a new blog post in my WordPress website about how to rank in Bing in 2026." Codex will ask for permission to access WordPress — click Allow. The free tier may take a minute or two, but you'll get a fully formatted, SEO-friendly blog post posted directly to your WordPress site.

Frequently Asked Questions

Is Codex free to use with WordPress?

Codex has a free tier with usage limits, and paid plans start at $8/month — significantly cheaper than Claude AI's $20/month starting plan. The WSP WordPress MCP plugin and MCP Adapter are both 100% free and open source.

What is Codex and how is it different from Claude AI?

Codex is an AI coding agent from OpenAI. Like Claude AI, it can connect to external tools via MCP and automate tasks through natural language. Codex is more accessible cost-wise (free tier + $8/month paid), while Claude AI ($20/month) generally produces higher-quality output. Both use the same WordPress MCP plugin.

Do I need coding skills to connect Codex with WordPress?

No coding skills required. You install two plugins, copy a config snippet into a TOML file, and generate an application password — the whole process takes about 5–10 minutes.

What can Codex do with my WordPress website?

Codex can create SEO-friendly blog posts, manage pages, handle WooCommerce products, update meta titles and descriptions, audit content for search rankings, and more — all through natural language instructions in the Codex chat window.

Is it safe to connect Codex to my WordPress site?

Yes, with appropriate precautions. The WSP WordPress MCP plugin gives you granular control — toggle off any ability you don't want Codex to use. Keep your application password private, and revoke it from WordPress at any time if needed.

What is MCP (Model Context Protocol)?

MCP (Model Context Protocol) is an open standard originally developed by Anthropic, now widely adopted by AI tools including Codex. It allows AI agents to securely connect with external systems — including your WordPress REST API — without direct server access.

Why does Codex take longer to respond than Claude AI?

On the free tier, Codex has rate limits that slow down processing. Paid Codex plans are faster. If speed is important, upgrading to a paid plan or switching to Claude AI will give you near-instant responses.

Can I use both Codex and Claude AI with the same WordPress MCP plugin?

Yes. The WSP WordPress MCP plugin works with both. The plugin's Config Files section provides separate configuration snippets for Claude AI (JSON format) and Codex (TOML format). You can connect both agents to the same WordPress site simultaneously.
